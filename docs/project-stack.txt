PROJECT-SPECIFIC ARCHITECTURE
--------------------------------------------------------------------------------

Game Components (/components/game/)

  - card-3d.tsx         - 3D hand card (unused, replaced by UI cards)

  - scene.tsx           - Main R3F scene orchestration
  - table.tsx           - Poker table and grid cells
  - ui.tsx              - HTML overlay (HUD, hand cards, info panels)
  - unit-3d.tsx         - Deployed card units on the field

Game Logic (/lib/)
  - store.ts            - Zustand game state (players, phases, combat)
  - card-data.ts        - Card generation and tag system

Custom Hooks (/hooks/)
  - use-mobile.tsx      - Mobile device detection

VISUAL STYLE
--------------------------------------------------------------------------------

90's Anime Aesthetic achieved through:
  - MeshToonMaterial   - Cel-shaded flat colors
  - Outlines           - Black borders on 3D objects
  - Bloom              - Glowing neon highlights
  - Scanlines          - CRT monitor effect
  - Noise              - Film grain texture

Color Palette:
  - Player 1: Cyan (#00d4ff)
  - Player 2: Pink (#ff66b2)
  - Gold accents: #ffd700
  - Table felt: #1a5a1a (dark green)

--------------------------------------------------------------------------------
GAME MECHANICS
--------------------------------------------------------------------------------

Resources:
  - DP (Deployment Points) - Currency to play cards
  - Max DP cap: 10
  - +1 Max DP each turn

Card Stats:
  - HP (Hit Points)
  - AD (Attack Damage)
  - DP (Cost)
  - Tags (up to 2)

Tags:
  - Air, Armor, Structure, Infantry, Stealth, Support, Heavy

Turn Structure:
  - Player 1 Phase (draw, deploy)
  - Player 2 Phase (draw, deploy)
  - End Phase (combat resolution)
